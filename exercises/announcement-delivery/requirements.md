# 要求仕様書：お知らせ配信機能

## 背景

社内向けの「お知らせ配信機能」を新規構築する。社内の全社員（約400名）に対して、人事部や総務部がお知らせを配信する。

現時点で確定している配信手段はメール1種類のみ。マーケ部の雑談で「将来は Slack や Teams にも流したいよね」「LINE WORKS を入れる話もあるかもしれない」という発言が出たが、時期も予算も責任者も決まっていない。経営層からの正式なオーダーでもない。

データの状態としても、現状でメール以外の宛先情報は社内に整備されていない。Slack ID や Teams ID をユーザーマスタに持たせる計画もまだない。

## 要求

1. 人事部・総務部の担当者がお知らせを作成・配信できる
2. 配信時に「全社員」「特定部署」「特定役職以上」などのセグメントを指定できる
3. お知らせは過去分も含めて社員ポータルで閲覧できる
4. 配信ログ（誰がいつ何を送ったか・配信成否）を残す

## 受け入れ基準

- 担当者がお知らせを作成・配信できる
- セグメント（全社員 / 特定部署 / 特定役職以上）を指定して配信できる
- 社員ポータルで過去のお知らせを閲覧できる
- 配信ログが残る

## 制約

- Spring Boot 3.x / Java 21
- 社内 SMTP サーバを使う
- ユーザーマスタは既存（`users` テーブルに `id`, `name`, `email`, `department`, `position` を持つ）

## 出力してほしいもの

- 実装ファイル一式
- 受け入れ基準のうち「特定部署を指定して配信できる」をカバーするテスト
