# 演習 09: 要求仕様書

## 背景

社内向けの「顧客マスタ管理画面」を新規構築する。営業部門が顧客情報を登録・参照・更新する。

顧客マスタは経理・営業・カスタマーサポートが共通で参照する基幹データなので、業務ルールがいくつかある。ただし今回のスコープでは「とりあえず登録・参照・更新ができる最小限の画面」を先に作りたい、というのが部長の意向。権限まわりや履歴は「次フェーズで」という整理。

ただし開発側として、最初から構造に織り込まないと後から困るものは織り込みたい。何を最初から織り込んで何を次フェーズに回すかは、設計判断としてあなたに委ねる。

## 要求

1. 顧客の登録・参照・更新ができる
2. 顧客情報は `customer_id`, `name`, `kana`, `phone`, `email`, `billing_address`, `shipping_address`, `credit_limit`, `payment_terms`, `is_active` を持つ
3. 重複登録（同一名 + 同一電話番号）を防ぐ
4. 営業部門の20名程度が同時に使う

## 受け入れ基準

- 顧客の登録・参照・更新ができる
- 同一名 + 同一電話番号の顧客は重複登録できない

## 制約

- Spring Boot 3.x / Java 21
- 顧客マスタは経理・営業・カスタマーサポートが共通で参照する
- 次フェーズで「権限管理」「変更履歴」「論理削除」「与信枠変更の承認フロー」を入れる予定（時期未定）

## 出力してほしいもの

- DB スキーマ
- 実装ファイル一式
- 受け入れ基準のうち「顧客の登録ができ、重複登録は弾かれる」をカバーするテスト
