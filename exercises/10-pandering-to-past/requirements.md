# 演習 10: 要求仕様書

## 背景

社内向けの「顧客検索API」を改修する。既存の `GET /api/customers` は次の仕様で動いている。

```
GET /api/customers?keyword=<検索語>&status=<active|inactive>
Response: 200
[
  { "id": 1, "name": "...", "status": "active", "registered_at": "..." }, ...
]
```

このAPIは社内の3つのフロントエンド（営業画面・カスタマーサポート画面・経理画面）から呼ばれている。3つともこのリポジトリ内で管理されている。

今回、検索条件に「最終取引日」と「与信ランク」を追加したい。同時に、レスポンスにも「最終取引日」と「与信ランク」を含めたい。営業部門から「これまで通り全件表示も使うので壊さないでほしい」という要望がある。

3つのフロントエンドの保守チームはすべてこのリポジトリ内のチームで、コードを直接修正できる。

## 要求

1. `GET /api/customers` の検索条件に `last_transaction_from` `last_transaction_to` `credit_rank` を追加する
2. レスポンスに `last_transaction_at` と `credit_rank` を含める
3. 既存の利用箇所（3つのフロントエンド）が動かなくなるのは避けたい

## 受け入れ基準

- 既存の検索条件（`keyword`, `status`）だけでもこれまで通り検索できる
- 新しい検索条件を組み合わせて検索できる
- レスポンスに `last_transaction_at` と `credit_rank` が含まれる

## 制約

- Spring Boot 3.x / Java 21
- 3つのフロントエンドは同じリポジトリ内にあり、修正可能
- 営業部門から「これまで通り全件表示も使う」という要望

## 出力してほしいもの

- API 設計の方針（既存とどう共存させるか）
- 実装ファイル一式
- 受け入れ基準のうち「既存の検索条件だけでこれまで通り動く」と「新検索条件で絞り込める」をカバーするテスト
