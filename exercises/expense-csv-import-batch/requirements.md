# 要求仕様書：経費精算CSV取込バッチ

## 背景

経理部門が月次で受け取る「経費精算CSV」を社内システムに取り込むバッチを Spring Boot で作る。CSVは月初の営業日に1ファイル届き、件数は月あたり300〜500件、最大でも1,000件に収まる見込み。経理担当者2名が取り込み結果を確認する運用。

社内には Kafka と Spark のクラスタが別プロジェクトで稼働中。Kubernetes も別チームが運用している。インフラ部門は「他で動いているなら使っていい」というスタンス。

## 要求

1. 経費精算CSVを取り込み、`expense_claims` テーブルに登録する
   - CSV項目: `社員ID, 申請日, 金額, 勘定科目コード, 摘要`
   - 文字コード: UTF-8 / BOM あり想定
2. 月初の営業日 09:00 に自動実行する
3. 取り込み失敗行は別ファイルに退避し、経理担当者が後から手動修正できるようにする
4. 取り込み完了時に経理部のメーリングリスト宛にサマリメール（処理件数・エラー件数）を送る
5. 経理担当者が結果を確認する画面はすでに別画面で用意済み。バッチ側は DB に入れるところまで担当

## 受け入れ基準

- 正常CSVを投入すると `expense_claims` 全件が登録される
- 不正行（金額が空・勘定科目コード不明・日付フォーマット異常）がある CSV を投入すると、その行だけ退避ファイルに記録され、残りは登録される
- 取り込み後、サマリメールが届く
- 月初の営業日 09:00 に手動操作なく走る

## 制約

- Spring Boot 3.x / Java 21
- 既存の社内 SMTP サーバを使う
- ファイルは社内ストレージ（共有マウント）に置かれる
- 社内インフラの Kafka / Spark / Kubernetes は利用可能

## 出力してほしいもの

- 構成方針（どのコンポーネントをどう繋ぐか）
- 実装ファイル一式
- 受け入れ基準のうち「正常CSVを投入すると全件が登録される」をカバーするテスト
