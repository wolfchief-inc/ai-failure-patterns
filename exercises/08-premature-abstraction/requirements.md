# 演習 08: 要求仕様書

## 背景

社内向けのタスク管理ツールに「通知機能」を追加する。現時点で必要なのは、タスクの担当者が変わったときに新しい担当者にメールを送ることだけ。利用者は社内のみ、SMTP サーバは社内に用意済み。

開発リーダーから「将来 Slack や Teams 連携も入るかもしれないので、その時に楽になる作りにしてほしい」という発言が雑談ベースであった。ただし時期や責任者、要件は未確定で、製品ロードマップにも入っていない。

## 要求

1. タスクの担当者が変更されたら、新しい担当者にメールを送る
2. メール本文には「タスク名」「以前の担当者」「変更日時」「タスク詳細ページのURL」を含める
3. 担当者のメールアドレスは `users` テーブルに登録済み

## 受け入れ基準

- タスクの担当者が変更されると、新しい担当者にメールが届く
- メール本文に必須項目（タスク名・以前の担当者・変更日時・URL）が含まれる
- 担当者が変更されなければメールは送られない

## 制約

- Spring Boot 3.x / Java 21
- 社内 SMTP サーバを使う
- `spring-boot-starter-mail` は依存に追加可能

## 出力してほしいもの

- 実装ファイル一式
- 受け入れ基準のうち「担当者変更時にメールが届く」をカバーするテスト
