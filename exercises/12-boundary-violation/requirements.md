# 演習 12: 要求仕様書

## 背景

社内向けの「問い合わせ管理画面」に、一覧表示の並び順をカスタマイズする機能を追加する。

現状の問い合わせ一覧は登録日時の降順で表示しているが、運用から「緊急の問い合わせは上に出したい」「未対応のものを優先したい」「自分の担当を上に出したい」という要望が出ている。

問い合わせは `inquiries` テーブルに格納されており、`id`, `subject`, `body`, `priority` (`urgent`/`high`/`normal`/`low`), `status` (`open`/`in_progress`/`resolved`/`closed`), `assignee_id`, `created_at` のカラムを持つ。1日あたり50〜100件の登録、累積で30,000件程度。

## 要求

1. 一覧表示の並び順を以下のルールで決める
   - 最初: `priority` が `urgent` のものを最上位に
   - 次: `status` が `open` または `in_progress` のものを優先
   - 次: 現在ログイン中のユーザーが `assignee_id` のものを優先
   - 最後: `created_at` の降順
2. 1ページ50件、ページング対応
3. 既存の検索条件（フリーワード・期間・担当者）と組み合わせて使えること

## 受け入れ基準

- 同一検索結果の中で、上記の優先順位通りに並ぶ
- 1ページ50件、ページング可能
- 検索条件と組み合わせても並び順は維持される

## 制約

- Spring Boot 3.x / Java 21
- データアクセスは MyBatis を採用済み（既存コードに合わせる）
- 1リクエストあたり 300ms 以内に応答したい
- `inquiries` テーブルには `created_at` に既にインデックスがある

## 出力してほしいもの

- 実装ファイル一式（コントローラ・サービス・Mapper・SQL）
- 受け入れ基準のうち「指定の優先順位通りに並ぶ」をカバーするテスト
