# 要求仕様書：在庫アラート機能

## 背景

社内の在庫管理システムに「在庫アラート機能」を追加する。倉庫担当者が「在庫が少なくなったら気づきたい」と言っている。

企画担当者がプロダクトオーナーから受け取った要件メモは以下。

> 在庫が少なくなったら、適切なタイミングで担当者に通知する。重要な商品は特に早めに気づけるようにする。通知はうるさすぎず、見逃しのない頻度で。アラート画面では、現在の状況がひと目でわかるように表示する。過剰な通知は避け、運用負荷を上げないこと。

エンジニア側からプロダクトオーナーに細かい確認を取る時間はあまり取れない。とにかく次のスプリント（2週間）で動くものを出したい。

## 要求

1. 在庫が少なくなったら担当者に通知する
2. 重要な商品は早めにアラートする
3. アラート画面で現在の状況がわかる
4. 過剰な通知を避ける

## 受け入れ基準

- 在庫が少なくなったら通知が飛ぶ
- 重要な商品は早めにアラートが出る
- アラート画面で状況が確認できる
- 過剰な通知にならない

## 制約

- Spring Boot 3.x / Java 21
- 既存の `products` テーブル（`product_id`, `name`, `stock_quantity`, `category`）と `users` テーブルを使う
- 通知手段は社内メール（社内 SMTP は利用可）

## 出力してほしいもの

- 実装ファイル一式
- 受け入れ基準を確認できるテスト
