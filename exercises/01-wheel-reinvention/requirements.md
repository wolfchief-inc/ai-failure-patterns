# 演習 01: 要求仕様書

> テストモード用。タグは付けない。背景・要求・受け入れ基準を厚めに記述したバージョン。受講者は必要に応じて要約してから貼り付けてもよい。

## 背景

社内の業務管理ツールを Spring Boot で新規構築している。利用者は同一社内ネットワーク内の数十名程度。外部公開予定はない。

認証基盤は別チームが OIDC で用意する計画があるが、初期リリースではメール+パスワードのフォーム認証で十分という合意がある。フレームワークとして Spring Boot 3.x / Java 21 を採用済み。Spring Security も依存に入っている。

## 要求

1. ログインAPIを実装する
   - POST `/api/login`
   - リクエスト: `{ "email": "string", "password": "string" }`
   - 認証成功時: 200。クライアントは以降のリクエストでこのユーザーとして扱われること
   - 認証失敗時: 401
2. 既存の `users` テーブルを使う
   - カラム: `id (BIGINT)`, `email (VARCHAR, UNIQUE)`, `password_hash (VARCHAR)`, `role (VARCHAR)`
   - パスワードは BCrypt でハッシュ済み
3. ログイン後、`/api/me` のような認証必須APIを叩くと当該ユーザーの `id` と `role` が取得できる
4. ログアウトAPI POST `/api/logout` も用意する

## 受け入れ基準

- 正しいメール+パスワードで `/api/login` を叩くと 200 が返り、続けて `/api/me` を叩くとログインしたユーザーの情報が返る
- 誤ったパスワードでは 401 が返る
- ログアウト後に `/api/me` を叩くと 401 が返る
- 認証必須APIに未ログインでアクセスすると 401 が返る

## 制約

- Spring Boot 3.x / Java 21
- 依存ライブラリの追加は不要な範囲で抑える
- 社内ネットワーク内のみで使う

## 出力してほしいもの

- 実装ファイル一式
- 主要な設定（必要なら）
- 受け入れ基準のうち1本をカバーする統合テスト
