# 演習 07: 要求仕様書

## 背景

総務部から「備品購入の社内申請をシステム化したい」という依頼が来た。現在は Excel + メール添付で運用しており、誰がどの段階で止めているか分からない、過去の申請履歴を探すのが大変、という不満がある。

総務の窓口担当者と1度ミーティングを持ったが、申請フロー（誰がどう承認するか）は「部門によって違うので、これから整理する」とのこと。承認者の権限、上長不在時の代理承認、却下後の再申請の扱いについては「現状は人によりまちまち」。承認後の発注作業は別システムなので今回はスコープ外。

来月の役員会で「進捗デモを見せたい」というオーダーが出ており、何かしら動くものを早めに見せたい。

## 要求

1. 申請者が「備品購入申請」を出せる（品目・数量・金額・理由）
2. 承認者が承認・却下できる
3. 申請者・承認者がそれぞれ自分関連の申請一覧を見られる
4. 申請の状態が変わったら申請者にメール通知する

## 受け入れ基準

- 申請者が申請を作成できる
- 承認者が承認・却下できる
- 申請者・承認者がそれぞれ自分関連の申請一覧を見られる
- 状態変化時にメールが届く

## 制約

- Spring Boot 3.x / Java 21
- 来月の役員会までに動くデモを見せたい

## 出力してほしいもの

- 主要な実装（コントローラ・サービス・モデル）
- DB スキーマ
- 受け入れ基準のうち「申請者が申請を作成し、承認者が承認できる」をカバーするテスト
