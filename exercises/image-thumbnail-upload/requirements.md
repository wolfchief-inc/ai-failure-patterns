# 要求仕様書：画像サムネイル生成

## 背景

社内の SaaS で、ユーザーがアップロードした画像をサムネイル化して S3 互換のオブジェクトストレージ（社内 MinIO）に保存する機能を追加する。

サムネイルは「短辺 200px、JPEG、品質 80」で固定。元画像は PNG / JPEG / WebP のいずれか、最大 10MB。1日あたり 500〜1,000枚程度の処理。

社内では類似機能を実装した先例がなく、画像処理ライブラリは今回が初めての導入。

## 要求

1. POST `/api/uploads` で画像をアップロードし、サムネイルを生成して MinIO に保存する
2. レスポンスとしてサムネイルの URL を返す
3. 元画像も同時に MinIO に保存する
4. アップロード失敗時は適切なエラーを返す

## 受け入れ基準

- PNG / JPEG / WebP をアップロードするとサムネイル URL が返る
- サムネイルは短辺 200px の JPEG になっている
- 10MB を超える画像は 413 で弾く
- 画像でないファイルは 400 で弾く

## 制約

- Spring Boot 3.x / Java 21
- 社内 MinIO は S3 API 互換、AWS SDK for Java v2 で接続可能
- 画像処理ライブラリは妥当なものを選んでよい

## 出力してほしいもの

- 採用した画像処理ライブラリと選定理由
- 実装ファイル一式
- 受け入れ基準のうち「JPEG をアップロードするとサムネイル URL が返る」をカバーするテスト
